## 1. 文档元数据标准

### 1.1 元数据块格式

所有文档顶部必须包含元数据块，采用 YAML 格式：

```yaml
---
title: [文档标题]
doc_version: [版本号，如 v1.0]
create_date: [创建日期，YYYY-MM-DD]
last_updated: [最后更新日期，YYYY-MM-DD，可选]
project: [项目名称]
domain: [专业领域，如 SCADA/PLC/仪表]
classification: [文档分类：需求/设计/规格/手册/规程]
---
```

### 1.2 文档分类定义

| 分类代码 | 名称 | 说明 | 示例 |
|----------|------|------|------|
| REQ | 需求文档 | 定义系统需求和功能要求 | 需求确认汇总、功能规格说明书 |
| DES | 设计文档 | 系统架构和详细设计 | 工艺流程设计、点位清单 |
| SPC | 规格文档 | 技术规格和接口定义 | 通讯接口规格书、位号编码规则 |
| OPR | 操作手册 | 操作指南和运维规程 | 操作手册、报警处理指南 |
| PRG | 程序文档 | PLC程序和SCADA组态说明 | 联锁逻辑说明书、程序规格书 |

## 3. Markdown 格式规范

### 3.1 标题层级

| 层级 | 语法 | 使用场景 |
|------|------|----------|
| H1 | `# ` | 文档标题，只有一个 |
| H2 | `## ` | 主章节 |
| H3 | `### ` | 二级子章节 |
| H4 | `#### ` | 三级子章节（慎用） |

**规则**：
- H1 标题与文件名保持一致
- 最多使用到 H4，避免过深嵌套
- 标题使用陈述性短语，避免"关于xxx"格式

### 3.2 标题首字母大写

| 正确 | 错误 |
|------|------|
| ## System Architecture | ## System architecture |
| ## Communication Protocols | ## Communication protocols |
| ## Emergency Stop Logic | ## emergency stop |

### 3.3 列表格式

**有序列表**：用于步骤和序列
```markdown
1. 第一步
2. 第二步
3. 第三步
```

**无序列表**：用于要点罗列
```markdown
- 关键要点一
- 关键要点二
  - 嵌套要点（缩进2空格）
  - 嵌套要点
```

**列表嵌套**：子列表缩进2空格
```markdown
1. 主步骤
   1.1 子步骤
   1.2 子步骤
2. 主步骤
```

### 3.4 代码块

**围栏代码块**（推荐）：
````markdown
```python
def read_sensor_data(sensor_id):
    """读取指定传感器数据"""
    return sensor_cache.get(sensor_id)
```
````

**行内代码**：使用反引号
```markdown
调用 `read_sensor_data()` 函数获取数据。
```

### 3.5 表格格式

```markdown
| 列1 | 列2 | 列3 |
|------|------|------|
| 值1 | 值2 | 值3 |
| 值4 | 值5 | 值6 |
```

**规则**：
- 表头与数据行用分隔线 `-`
- 分隔线至少3个字符
- 列对齐：默认左对齐，数字右对齐

### 3.6 链接格式

**内部链接**：使用相对路径
```markdown
参见 [通讯接口规格书](../documents/通讯接口规格书.md)
```

**外部链接**：使用完整URL
```markdown
详情见 [IEC 61131-3](https://www.iec.org)
```

## 4. 内容编写规范

### 4.1 术语使用

**术语表**（本项目统一术语）：

| 术语 | 定义 | 避免使用 |
|------|------|----------|
| SCADA | 数据采集与监控系统 | Scada、scada |
| PLC | 可编程逻辑控制器 | plc、可编程控制器 |
| 位号 | 数据点的唯一标识符 | Tag、点位 |
| I/O | 输入/输出 | io、IO模块 |
| 联锁 | 基于安全条件的自动控制动作 | 互锁、联动 |
| ESD | 紧急停车系统 | 急停系统 |
| UHT | 超高温瞬时杀菌 | 超高温杀菌 |
| CIP | 就地清洗系统 | 清洗系统 |
| HMI | 人机界面 | 操作界面 |
| OPC UA | 开放平台通信统一架构 | OPC |

### 4.2 语言风格

| 规则 | 正确示例 | 错误示例 |
|------|----------|----------|
| 使用主动语态 | 系统**发送**数据到PLC | 数据**被发送**到PLC |
| 使用现在时 | 系统**检测**到异常 | 系统**将检测**到异常 |
| 使用第二人称 | **您**可以配置参数 | **用户**可以配置参数 |
| 简洁表达 | **如果**发生故障，**则**停止 | **当**故障**的时候**，**那么**就**会**停止 |
| 明确主语 | PLC **控制器** 发出指令 | **发出**指令（主语不明） |

### 4.3 数值和单位

| 类型 | 格式 | 示例 |
|------|------|------|
| 温度 | 数字 + 单位（无空格） | 121℃、-20℃ |
| 压力 | 数字 + 单位 | 0.6MPa、100kPa |
| 流量 | 数字 + 单位 | 5000L/h、10m³/h |
| 百分比 | 数字 + % | 95%、≤0.5% |
| 范围 | 范围用 ~ 或 - | 10~20℃、100-200kPa |

### 4.4 时间表达

| 类型 | 格式 | 示例 |
|------|------|------|
| 日期 | YYYY-MM-DD | 2026-04-28 |
| 时间 | HH:MM:SS（24小时制） | 14:30:00 |
| 日期时间 | YYYY-MM-DD HH:MM | 2026-04-28 14:30 |

## 5. 图表规范

### 5.1 ASCII 图表

使用等宽字体绘制，保持对齐：

```text
┌─────────────────────┐
│      设备名称        │
├─────────────────────┤
│  参数1: 值          │
│  参数2: 值          │
└─────────────────────┘
```

### 5.2 图表标题

所有图表必须有编号和标题：

```markdown
**图 1: 系统架构图**

**表 1: 通讯配置参数**
```

## 6. AI 生成文档规范

### 6.1 AI 生成 Prompt 模板

当使用AI辅助生成文档时，使用以下Prompt结构：

```markdown
## 任务
[生成/更新/补充] [文档类型]，主题：[具体内容]

## 上下文
- 项目：[项目名称]
- 文档类型：[REQ/DES/SPC/OPR/PRG]
- 已有章节：[章节列表]
- 目标读者：[开发人员/运维人员/客户]

## 参考文档
- [相关文档1]：描述
- [相关文档2]：描述

## 要求
1. 严格遵循本规范的格式和结构
2. 使用本项目统一的术语表
3. 技术参数必须与参考文档一致
4. [其他特定要求]

## 输出格式
[期望的输出格式]
```

### 6.2 AI 生成质量检查清单

生成后需逐项检查：

- [ ] 术语使用符合术语表，无自创术语
- [ ] 数据参数与相关文档一致，无矛盾
- [ ] 格式符合本规范要求
- [ ] 无歧义表述，主语明确
- [ ] 代码块已声明语言
- [ ] 链接路径正确（相对路径或完整URL）
- [ ] 表格格式正确，表头有分隔线
- [ ] 标题层级正确（H1只有一个）

### 6.3 禁止事项

- **禁止**预发布未来功能或未实现的设计
- **禁止**在规格文档中使用"可能"、"也许"等模糊词汇
- **禁止**技术参数留空或标注"TBD"（应标注实际值或说明待定原因）
- **禁止**截断代码块或命令示例

## 7. 文档评审流程

### 7.1 评审级别

| 文档类型 | 评审要求 | 评审人 |
|----------|----------|--------|
| REQ（需求） | 必须评审 | 产品经理 + 技术负责人 |
| DES（设计） | 必须评审 | 技术负责人 + 专家 |
| SPC（规格） | 必须评审 | 技术负责人 + 实施工程师 |
| OPR（操作） | 建议评审 | 运维 + 最终用户代表 |
| PRG（程序） | 必须评审 | PLC工程师 + 测试工程师 |

### 7.2 评审检查点

1. **完整性**：是否覆盖所有必要章节
2. **一致性**：与其他相关文档是否矛盾
3. **可执行性**：规格文档是否可指导实施
4. **可读性**：术语是否统一，表述是否清晰

## 9. 文件命名规范

### 9.1 命名规则

```
[类型]_[文档名称].[扩展名]

示例：
SPC_通讯接口规格书.md
REQ_需求确认汇总.md
DES_系统架构设计.md
OPR_操作手册.md
PRG_联锁逻辑说明书.md
```

### 9.2 命名禁止

- 禁止使用中文标点符号
- 禁止使用空格
- 禁止使用超过50个字符
- 禁止使用特殊字符（仅允许 `_` 和 `-`）
