# 代码重构技能

安全、系统地重构代码，提升可维护性。

## 重构原则

### 前提条件
```
1. 有测试覆盖
2. 在版本控制下
3. 理解原有逻辑
```

### 重构步骤
```
1. 识别坏味道
2. 制定重构计划
3. 小步修改
4. 频繁提交
5. 验证测试通过
```

## 常见坏味道

| 坏味道 | 特征 | 重构手法 |
|-------|------|---------|
| **重复代码** | 复制粘贴3次以上 | 提取函数/类 |
| **过长函数** | 超过50行 | 提取函数 |
| **过大类** | 超过300行 | 拆分职责 |
| **过长参数** | 参数超过3个 | 引入参数对象 |
| **霰弹式修改** | 改一个功能改多个文件 | 移动函数/类 |
| **特性羡慕** | 类总用另一个类的数据 | 移动方法 |
| **数据泥团** | 总是同时出现的数据 | 提取类 |

## 重构手法速查

### 提取
```
提取函数：长函数 → 多个小函数
提取类：多重职责 → 单一职责
提取变量：复杂表达式 → 有名变量
```

### 重命名
```
变量：描述意图
函数：动词短语
类：名词
```

### 移动
```
移动方法：类间转移职责
移动字段：改变所有权
```

### 简化
```
简化条件：合并重复条件表达式
简化函数：移除不必要的参数
简化类：提取接口
```

## 安全重构 checklist

- [ ] 有测试覆盖？
- [ ] 一次只做一个重构？
- [ ] 每步都运行测试？
