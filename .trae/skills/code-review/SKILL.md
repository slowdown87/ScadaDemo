# 代码审查技能

系统性审查代码，确保质量、安全和可维护性。

## 审查维度

| 维度 | 关注点 |
|-----|-------|
| **正确性** | 逻辑错误、边界条件、空指针 |
| **安全性** | SQL注入、XSS、敏感信息、权限控制 |
| **性能** | 时间复杂度、内存泄漏、资源释放 |
| **可维护性** | 代码重复、命名规范、注释 |
| **最佳实践** | 设计模式、语言特性、框架使用 |

## 审查流程

```
1. 理解代码上下文
   - 这是什么模块/功能？
   - 关键输入输出是什么？

2. 静态分析
   - 代码结构扫描
   - 明显问题标记

3. 深度审查
   - 业务逻辑验证
   - 边界条件检查
   - 安全漏洞扫描

4. 审查报告
   - 问题列表（严重程度）
   - 改进建议
   - 优先修复顺序
```

## 问题严重程度

| 级别 | 标记 | 说明 |
|-----|------|------|
| 严重 | 🔴 | 必须修复，安全/致命Bug |
| 中等 | 🟡 | 应该修复，性能/逻辑问题 |
| 建议 | 🟢 | 可以改进，可读性/优化 |

## 输出格式

```
## 代码审查报告

### 🔴 严重问题
- [文件:行号] 问题描述
  - 影响：...
  - 建议：...

### 🟡 中等问题
- [文件:行号] 问题描述
  - 影响：...
  - 建议：...

### 🟢 改进建议
- [文件:行号] 建议描述
  - 原因：...

### 总结
- 总问题数：X
- 严重：X | 中等：X | 建议：X
```

## 常用检查项

### 安全检查
- [ ] 敏感信息硬编码？
- [ ] 用户输入验证？
- [ ] SQL/命令拼接？
- [ ] 权限检查？
- [ ] 加密存储？

### 性能检查
- [ ] 循环内查询数据库？
- [ ] 大数据量加载到内存？
- [ ] 连接/资源未释放？
- [ ] 重复计算？

### 代码质量
- [ ] 命名清晰？
- [ ] 函数过长（>50行）？
- [ ] 重复代码？
- [ ] 缺少注释？
