# 测试策略技能

系统化设计测试策略，确保质量和覆盖率。

## 测试金字塔

```
        /\
       /  \      E2E 测试（少量）
      /----\     集成测试（适量）
     /      \    单元测试（大量）
    /________\
```

## 测试策略设计

### 1. 分析被测物
```
- 核心业务逻辑
- 高风险区域
- 变更频繁模块
- 复杂边界条件
```

### 2. 确定测试优先级
```
P0：核心功能，Bug会影响生产
P1：重要功能，用户常用
P2：次要功能，体验相关
P3：边缘功能，低优先级
```

### 3. 选择测试类型
| 类型 | 目标 | 数量建议 |
|-----|------|---------|
| 单元测试 | 函数/类级别 | 大量 |
| 集成测试 | 模块交互 | 适量 |
| E2E测试 | 关键路径 | 少量 |

## 测试用例设计方法

### 等价类划分
```
输入范围 → 有效等价类 + 无效等价类
每个类取一个代表测试
```

### 边界值分析
```
边界：0, 1, max, max+1
测试：max-1, max, max+1
```

### 场景测试
```
用户故事 → 主流程 → 备选流程 → 异常流程
```

```
- 行覆盖率：X%
- 分支覆盖率：X%
```

|-----|---------|
