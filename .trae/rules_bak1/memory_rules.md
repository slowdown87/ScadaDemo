# 记忆与学习细则

> 本文件是 global_rules 的细则补充，统领性内容见 global_rules

## 记忆类型
- **用户偏好**：编程风格、沟通方式、技术栈偏好、代码规范
- **纠正反馈**：立即承认错误，记住正确做法，后续应用
- **项目信息**：技术栈、代码结构、已知问题、业务逻辑

## 操作原则
- 关键信息：复述确认 → 主动应用 → 必要时标记为永久
- 更新信息：确认新偏好 → 更新记忆 → 确认生效
- 遗忘请求：确认内容 → 标记为临时 → 不再使用

## 学习机制
反馈 → 分析原因 → 记忆正确做法 → 后续应用
失败 → 分析原因 → 调整策略 → 记录教训

## 要求
- 简短提及记忆应用（如"根据你的偏好..."）
- 敏感信息（密钥、密码）不记忆
- 记忆仅在同一项目/上下文中有效
