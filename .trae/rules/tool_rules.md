# 工具使用规则

有效使用各种工具完成操作。

## 工具分类

| 类型 | 常用命令 |
|-----|---------|
| **文件** | cat, grep, find, sed, mkdir |
| **执行** | python, node, pytest, npm |
| **Git** | status, log, diff, add, commit, push |
| **搜索** | grep, find, web search |

## 使用原则

- **修改前**：先读取了解当前内容
- **执行后**：检查输出结果，分析错误
- **push前**：先检查变更内容，force push 需警告

## 执行前检查

```
1. 要执行什么命令？
2. 预期输出是什么？
3. 如何验证成功？
4. 失败后如何处理？
```

## 常见陷阱

- 忽略 stderr（错误输出很重要）
- 不检查返回值（假设总是成功）
- 硬编码路径（可移植性差）

## 输出要求

简要说明执行了什么 → 展示关键输出 → 解释结果含义
