# 技能与协作

## 技能选择

| 条件 | 技能 | 说明 |
|------|------|------|
| Bug/Issue | swe-agent-mode | 自动修复模式 |
| 子任务>3/需执行顺序 | planning | 任务规划技能 |
| 安全/认证/敏感数据 | security | 安全审计技能 |
| 代码修改 | code-review | 代码审查技能 |
| 深度分析/新技术 | guided-ai | 引导性AI技能 |
| 复杂多视角决策 | multi-agent-sim | 多角色模拟技能 |
| 架构设计/技术选型 | architecture | 架构设计技能 |
| 出错/不工作 | debug | 调试技能 |
| 重构代码 | refactoring | 重构技能 |
| 设计测试 | testing | 测试策略技能 |
| 写文档 | documentation | 文档编写技能 |
| 创建新技能 | skill-creator | 技能创建技能 |

## 常用组合

| 场景 | 技能组合 |
|------|----------|
| 新功能 | planning → swe-agent-mode → testing → code-review |
| Bug修复 | debug → swe-agent-mode → testing |
| 重构 | code-review → refactoring → testing |
| 架构决策 | architecture → guided-ai |

## 激活流程

### 激活步骤

| # | 步骤 | 说明 |
|---|------|------|
| 1 | 标注激活 | [激活：xxx] |
| 2 | 执行模板 | 按技能模板执行 |
| 3 | 完成自检 | 完成自检清单 |
| 4 | 输出结果 | 输出结果 |

### 技能选择检查清单

```
[ ] 识别任务类型
[ ] 选择合适技能

[ ] 如不适用，回退到默认流程
```

## 规划执行铁律

| 错误行为 | 正确行为 |
|----------|----------|
| 规划做20%就进入执行 | 必须完成规划检查清单才能执行 |
| 只做框架丢三落四 | 每个子任务必须有输入/输出/风险/验证 |
| 做完就认为差不多了 | 必须整体验证各部分协调一致 |

未通过自检清单 → 不输出计划 → 不进入执行 → 列出缺失项等待补充

## 自主决策范围

### AI可自行决定

| 场景 | 说明 |
|------|------|
| 简单重构 | 单函数提取/重命名 |
| 单文件修改 | <50行 |
| 常规调试 | 添加日志/打印变量 |
| 代码审查 | 仅提供报告 |

### 必须用户确认

| 场景 | 说明 |
|------|------|
| 多文件修改 | 涉及多个文件 |
| 删除操作 | 删除文件或目录 |
| 不可逆操作 | 无法回退的操作 |

## 动态协作

### 技能组合

| 场景 | 技能组合 |
|------|----------|
| 复杂调试 | debug + planning |
| 架构决策 | architecture + guided-ai |

### 切换规则

| 遇到障碍 | 切换到 |
|-----------|--------|
| 需要系统性规划 | planning |
| 发现安全风险 | security |
| 需要验证 | testing |

## 判断标准

| 场景 | 简单 | 复杂 |
|------|------|------|
| 重构 | 单函数提取/重命名 | 跨文件职责重组 |
| 修改 | <50行/单文件 | >50行/多文件 |
| 调试 | 添加日志/打印变量 | 跨进程/并发问题 |
| 审查 | 单文件/<200行 | 多文件/核心模块 |
