# 工具、项目与记忆

## 工具选择

| 场景 | 推荐工具 | 说明 |
|------|----------|------|
| 读取文件 | Read | 内置工具，更稳定 |
| 执行命令 | RunCommand | 注意blocking参数 |
| 搜索代码 | Grep / SearchCodebase | Grep精确，Search模糊 |
| 批量查找 | Glob + Grep | 先定位目录再搜索 |
| 修改文件 | SearchReplace | 精准替换，不丢内容 |
| 创建文件 | Write | 仅在必要时使用 |
| 跟踪任务 | TodoWrite | 复杂任务必备 |

## 命令对照

| 类型 | Linux | Windows |
|------|-------|---------|
| 文件读取 | cat | Get-Content |
| 内容搜索 | grep | Select-String |
| 文件查找 | find | Get-ChildItem -Recurse |

优先使用 Trae IDE 内置工具，更稳定，权限更安全

## 工具原则

| 原则 | 说明 |
|------|------|
| 修改前先读取 | 避免误操作 |
| 执行后检查输出 | 及时发现错误 |
| push前检查变更 | 防止意外提交 |
| 跨平台兼容 | 避免硬编码路径 |

## 效率原则

| 原则 | 说明 | 限制 |
|------|------|------|
| 并行执行 | 独立任务可并行 | 最多3个 |
| 批量验证 | 先验证单次再全量 | - |
| 脚本测试 | 复杂脚本先测试 | - |

## 项目管理

### 核心铁律

1. 无完整设计不编码 → 设计文档需在 \docs\ 目录
2. 无规范不开发 → 技术规范需在 \docs\ 目录
3. 无计划不执行 → 计划通过状态快照.json校验
4. 文档先行 → 文档位置：\docs\[模块名].md
5. 跳步强制拦截 → AI自动拦截 + 提示用户确认

### 跳步拦截触发条件

| 场景 | AI动作 |
|------|--------|
| 缺少设计文档 | 停止 + 输出"缺少设计文档，是否补充" |
| 缺少计划 | 停止 + 提示创建任务列表 |
| 跳过文档直接编码 | 停止 + 说明违反的铁律 |

### 自主职责

| 职责 | 说明 |
|------|------|
| 优先局部读取 | 避免全量读取浪费时间 |
| 会话开始同步 | 确认当前进度和状态 |
| 变更先评审 | 需求/架构/设计变更必须评审 |

## 状态一致性管理

### 核心原则

状态快照.json是项目状态的唯一真相源

### 状态快照更新时机

| 时机 | 触发条件 | 操作 |
|------|----------|------|
| 任务开始 | 收到明确任务 | 创建/更新快照 |
| 规划完成 | planning技能激活后 | currentPhase="规划" |
| 执行开始 | 开始实施 | currentPhase="执行" |
| 阶段切换 | 规划→执行→验证→完成 | 更新currentPhase |
| 任务完成 | 达成目标 | task.status="完成" |

### 操作规范

规划时由planning技能生成和更新，详见 [planning技能文档](../skills/planning/SKILL.md)

### 禁止行为

| 禁止 | 说明 |
|------|------|
| 更新不同步 | 更新Phase但不同步状态快照 |
| 无快照执行 | 没有状态快照直接进入执行阶段 |

## 任务执行规范

### 执行前准备

- 读取状态快照，确认任务上下文
- 读取规划文档，确认执行计划

### 执行监控

| 活动 | 规则 |
|------|------|
| 跟踪进度 | 参照状态快照的task.progress |
| 实施细节 | 参照规划文档的子任务列表 |
| 记录决策 | 关键决策点记录到规划文档 |

### 状态快照更新

| 时机 | 操作 |
|------|------|
| 子任务完成 | progress += 1 |
| 发现阻塞 | blockers添加原因 |
| 阻塞消除 | blockers移除原因 |
| 需要确认 | pendingConfirmations添加项 |
| 确认完成 | pendingConfirmations移除项 |

### 执行后

- 更新状态快照的lastUpdated
- 更新状态快照的currentPhase（如进入下一阶段）

## 上下文交接

### 需要记录的信息

| 类型 | 说明 |
|------|------|
| 任务进度 | 已完成/进行中/未开始 |
| 未解决问题 | 技术难点/依赖项 |
| 待确认事项 | 需要用户决策 |
| 项目约束 | 时间/资源/技术限制 |

### 上下文模板

```
当需要交接或记录状态时，使用以下格式：

| 类型 | 格式 |
|------|------|
| 任务进度 | 已完成/进行中/未开始 |
| 未解决问题 | 技术难点/依赖项 |
| 待确认事项 | 需要用户决策 |
| 项目约束 | 时间/资源/技术限制 |

模板：
当前状态
- **任务**：xxx
- **进度**：xxx/xxx
- **阻塞**：xxx
- **待确认**：xxx
```

## 记忆与学习

| 类型 | 说明 | 敏感信息 |
|------|------|----------|
| 用户偏好 | 沟通风格/习惯 | 不记忆 |
| 纠正反馈 | 用户纠正的内容 | 不记忆 |
| 项目信息 | 架构/规范/约束 | 不记忆密钥 |

敏感信息永不记忆：API Key、Token、密码、credentials

### 记忆层

记忆层位于 .trae\memory\ 目录，用于持久化存储跨会话信息：

| 类型 | 路径 | 用途 |
|------|------|------|
| 语义记忆 | memory\semantic\ | 结构化事实和项目知识 |
| 情景记忆 | memory\episodic\ | 会话历史和关键事件 |
| 程序记忆 | memory\procedural\ | 最佳实践和工作流程 |

详见 [记忆层设计文档](../memory/semantic/user_preferences.md)

## 经验积累

### 激励机制

完成任务后，将经验存入记忆层：

| 完成情况 | 存入位置 | 内容 |
|----------|----------|------|
| 任务顺利完成 | 程序记忆 | 成功的流程模式 |
| 遇到问题并解决 | 程序记忆 | 问题解决模式 |
| 决策被用户肯定 | 程序记忆 | 有效决策模式 |
| 良好交互风格 | 用户偏好 | 沟通风格偏好 |

### 积累触发条件

- 任务完成后 → 主动询问是否记录经验
- 用户明确肯定 → 立即记录
- 遇到新问题并解决 → 解决后记录解决方案

### 记录格式

```markdown
## 经验记录

**日期**：YYYY-MM-DD
**类型**：流程|问题解决|决策|沟通
**任务**：任务名称
**经验**：具体经验内容
**效果**：带来的好处
```

## 性能与资源管理

| 类型 | 控制 | 说明 |
|------|------|------|
| 简单问答 | <200字 | 简洁直接 |
| 方案分析 | <500字 | 分层递进 |
| 复杂报告 | 分段<500字 | 每段独立可读 |
| 工具调用 | 单任务<10次 | 避免循环 |
| 循环迭代 | 最多5次 | 达上限必须汇报 |
