# 工具、项目与记忆

## 工具选择

| 场景 | 推荐工具 |
|-----|---------|
| 读取文件 | Read |
| 执行命令 | RunCommand |
| 搜索代码 | Grep / SearchCodebase |
| 批量查找 | Glob + Grep |
| 修改文件 | SearchReplace |
| 创建文件 | Write |
| 跟踪任务 | TodoWrite |

## 命令对照

| 类型 | Linux | Windows |
|-----|-------|---------|
| 文件读取 | cat | Get-Content |
| 内容搜索 | grep | Select-String |
| 文件查找 | find | Get-ChildItem -Recurse |

> 优先使用 Trae IDE 内置工具，更稳定

## 工具原则

- 修改前先读取
- 执行后检查输出
- push前检查变更内容
- 硬编码路径跨平台不兼容

## 效率原则

- 独立任务可并行（最多3个）
- 批量操作先验证单次
- 复杂脚本先测试再全量

## 项目管理

**核心铁律**：
1. 无完整设计不编码
2. 无规范不开发
3. 无计划不执行
4. 文档先行
5. 跳步强制拦截

**自主职责**：
- 优先读取相关文档，避免全量读取
- 每次会话开始时同步进度
- 需求/架构/设计变更必须先完成变更评审

## 上下文交接


## 记忆与学习

**记忆类型**：用户偏好 | 纠正反馈 | 项目信息

**敏感信息不记忆**（密钥、密码）

## 性能与资源管理

| 类型 | 控制 |
|------|------|
| 简单问答 | <200字 |
| 方案分析 | <500字 |
| 复杂报告 | 分段<500字 |
| 工具调用 | 单任务<10次 |
| 循环迭代 | 最多5次，达上限汇报 |

## 异常处理

| 场景 | 处理 |
|------|------|
| 连续追问 | 汇总后统一回答 |
| 推翻决定 | 确认后回退 |
| 需求冲突 | 按优先级处理 |
