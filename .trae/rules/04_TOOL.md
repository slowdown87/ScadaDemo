# 工具、项目与记忆

| 类型 | Linux | Windows |

> 优先使用 Trae IDE 内置工具，更稳定

## 工具原则

- 修改前先读取
- 执行后检查输出
- push前检查变更内容
- 硬编码路径跨平台不兼容

## 效率原则

- 独立任务可并行（最多3个）
- 批量操作先验证单次
- 复杂脚本先测试再全量

## 项目管理

**核心铁律**：
1. 无完整设计不编码
2. 无规范不开发
3. 无计划不执行
4. 文档先行
5. 跳步强制拦截

**自主职责**：
- 优先读取相关文档，避免全量读取
- 每次会话开始时同步进度
- 需求/架构/设计变更必须先完成变更评审

### 状态一致性管理

**核心原则**：状态快照.json是项目状态的唯一真相源

**强制更新场景**（满足任一即触发）：
1. 规划完成（任何技能激活后）
2. Phase/阶段切换
3. 重大决策或方案变更
4. 阻塞项新增或消除
5. 任务进度变更

**操作步骤**：
1. 打开 `状态快照.json`
2. 更新任务进度、当前阶段、阻塞项
3. 检查 `升级方案.md` 与快照一致
4. 如不一致 → 立即修复

**禁止行为**：
- 更新Phase进度但不同步状态快照
- 没有状态快照直接进入执行阶段

## 记忆与学习

**记忆类型**：用户偏好 | 纠正反馈 | 项目信息

**敏感信息不记忆**（密钥、密码）

## 上下文管理

TodoWrite跟踪实时热点，状态快照.json记录阶段性状态，超长对话（>10轮或>500字）主动摘要。

## 性能与资源管理

| 类型 | 控制 |
|------|------|
| 简单问答 | <200字 |
| 方案分析 | <500字 |
| 复杂报告 | 分段<500字 |
| 工具调用 | 单任务<10次 |
| 循环迭代 | 最多5次，达上限汇报 |

## 异常处理

| 场景 | 处理 |
|------|------|
| 连续追问 | 汇总后统一回答 |
| 推翻决定 | 确认后回退 |
| 需求冲突 | 按优先级处理 |
